# XFS Disk Structure (磁盘结构)

## 一、XFS磁盘布局概述

### 1.1 整体布局

XFS将磁盘空间划分为多个独立的管理单元，支持大容量和高并发：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              XFS Disk Layout                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  +-----------+-----------+-----------+-----------+-----------+-----------+  │
│  |   AG 0    |   AG 1    |   AG 2    |   AG 3    |   AG 4    |   ...     |  │
│  +-----------+-----------+-----------+-----------+-----------+-----------+  │
│                                                                             │
│  每个AG大小相等，独立管理空间和inode                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 关键特性

| 特性 | 说明 |
|------|------|
| 分配组 (AG) | 将磁盘划分为独立管理单元 |
| B+树索引 | 所有元数据使用B+树索引 |
| Extent分配 | 使用extent而非块链 |
| 日志记录 | 元数据操作先写日志 |
| 实时设备 | 可选实时设备存储大文件 |

## 二、AG内部结构

### 2.1 AG详细布局

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            Allocation Group (AG)                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  Sector 0:   ┌─────────────────────────────────────────────────────────┐   │
│              │                    Super Block (SB)                      │   │
│              │                    512 bytes                              │   │
│              │                    文件系统范围信息                        │   │
│              └─────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Sector 1:   ┌─────────────────────────────────────────────────────────┐   │
│              │                    AGF (AG Free space)                    │   │
│              │                    空间分配信息                            │   │
│              │                    BNO/CNT B+树根                          │   │
│              └─────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Sector 2:   ┌─────────────────────────────────────────────────────────┐   │
│              │                    AGI (AG Inode info)                    │   │
│              │                    inode分配信息                          │   │
│              │                    INO/FINO B+树根                        │   │
│              └─────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Sector 3:   ┌─────────────────────────────────────────────────────────┐   │
│              │                    AGFL (AG Free List)                    │   │
│              │                    空闲块列表                              │   │
│              │                    支持快速分配                            │   │
│              └─────────────────────────────────────────────────────────┘   │
│                                                                             │
│  Data Area:  ┌─────────────────────────────────────────────────────────┐   │
│              │                    Data Blocks                           │   │
│              │                    用户数据区域                            │   │
│              │                    文件内容、目录、属性等                  │   │
│              └─────────────────────────────────────────────────────────┘   │
│                                                                             │
│  B+tree Area:┌─────────────────────────────────────────────────────────┐   │
│              │                    B+tree Nodes                         │   │
│              │                    各种B+树节点                            │   │
│              │                    空间管理、inode索引等                  │   │
│              └─────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 AG头部结构顺序

1. **超级块 (Superblock)**: AG 0包含主超级块，其他AG包含备份
2. **AGF (Allocation Group Free space)**: 空闲空间管理
3. **AGI (Allocation Group Inode)**: inode管理
4. **AGFL (Allocation Group Free List)**: 空闲块列表

## 三、超级块结构

### 3.1 主超级块位置

- AG 0的 Sector 0 (512字节偏移)
- 魔数: "XFSB" (0x58465342)

### 3.2 备份超级块

每个AG的 Sector 0都包含超级块，但只有AG 0的主超级块被使用。

### 3.3 关键字段

```c
struct xfs_sb {
    __be32  sb_magicnum;           /* 魔数: XFSB */
    __be32  sb_blocksize;          /* 块大小 (bytes) */
    __be64  sb_dblocks;            /* 数据块总数 */
    __be64  sb_rblocks;            /* 实时设备块数 */
    __be64  sb_rextents;           /* 实时设备extents */
    __be64  sb_logstart;           /* 日志开始块 */
    __be64  sb_rootino;            /* 根目录inode */
    __be64  sb_rbmino;             /* 根目录位图inode */
    __be64  sb_rsumino;            /* 实时摘要inode */
    __be32  sb_rextsize;           /* 实时extent大小 */
    __be32  sb_agblocks;           /* 每个AG块数 */
    __be32  sb_agcount;            /* AG总数 */
    __be32  sb_rbmblocks;          /* 实时位图块数 */
    __be32  sb_logblocks;          /* 日志块数 */
    /* ... 更多字段 ... */
};
```

## 四、B+树索引系统

### 4.1 B+树类型

| B+树类型 | 用途 | 位置 |
|----------|------|------|
| BNO B+树 | 按起始块号索引空闲空间 | AGF |
| CNT B+树 | 按连续块数索引空闲空间 | AGF |
| INO B+树 | 按inode号索引已分配inode | AGI |
| FINO B+树 | 按inode号索引空闲inode | AGI |
| RMAP B+树 | 反向映射 (块→所有者) | AGF |
| REFC B+树 | 引用计数 | AGF |

### 4.2 B+树节点结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            B+tree Node                                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 节点头 (xfs_btree_block)                                             │   │
│  │  - magic: 魔数标识                                                    │   │
│  │  - level: 树层级 (0=叶节点)                                           │   │
│  │  - numrecs: 记录数量                                                  │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │ 键值对数组                                                            │   │
│  │  [key1, ptr1], [key2, ptr2], ...                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 五、数据区域组织

### 5.1 Extent分配

XFS使用extent而非块链分配空间：

```
struct xfs_bmbt_rec {
    __be64  l0;  /* 起始块号 + 标志 */
    __be64  l1;  /* 块数 + 起始偏移 */
};
```

### 5.2 数据fork格式

| 格式 | 说明 | 存储方式 |
|------|------|----------|
| 本地格式 | 数据直接存储在inode中 | inode内 |
| Extent格式 | 使用extent列表 | extent指针 |
| B+树格式 | 使用B+树索引extent | B+树根 |

### 5.3 Inode数据区域

inode可以包含：
1. **数据fork**: 文件内容或目录项
2. **属性fork**: 扩展属性
3. **扩展数据**: 内联数据或指针

## 六、实时设备支持

### 6.1 实时设备概念

XFS支持将大文件存储在单独的"实时设备"上：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    XFS with Realtime Device                                │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  +---------------------------+ +-----------------------------------------+  │
│  |     Data Device           | |         Realtime Device                |  │
│  |  (常规文件系统)            | |  (大文件存储)                           |  │
│  |                           | |                                         |  │
│  |  - 元数据                 | |  - 大文件数据                           |  │
│  |  - 小文件                 | |  - 连续分配                             |  │
│  |  - 目录                   | |                                         |  │
│  +---------------------------+ +-----------------------------------------+  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 6.2 实时设备结构

- 实时位图: 跟踪实时设备空间分配
- 实时摘要: 空间使用统计
- 固定extent大小: 提高连续分配性能
