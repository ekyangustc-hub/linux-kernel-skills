# XFS Runtime File Deletion (运行时文件删除)

## 一、概述

### 1.1 文件删除流程

XFS文件删除是一个多阶段的原子操作：

```
用户空间: unlink("file") 或 rmdir("dir")
    ↓
VFS层: vfs_unlink() 或 vfs_rmdir()
    ↓
XFS层: xfs_remove()
    ↓
事务开始 → 目录项删除 → 链接计数减1 → 资源释放 → 事务提交
```

### 1.2 删除类型

| 删除类型 | 系统调用 | XFS函数 | 特殊处理 |
|----------|----------|----------|----------|
| 文件删除 | unlink() | xfs_remove() | 硬链接检查 |
| 目录删除 | rmdir() | xfs_rmdir() | 空目录检查 |
| 强制删除 | unlink() | xfs_remove() | 忽略打开文件 |
| 递归删除 | N/A | xfs_remove() + 遍历 | 用户空间处理 |

### 1.3 删除阶段

```
阶段1: 验证阶段
  - 权限检查
  - 存在性检查
  - 类型检查（目录非空等）

阶段2: 元数据删除
  - 目录项删除
  - 链接计数更新
  - 时间戳更新

阶段3: 资源释放
  - inode释放（如果链接数=0）
  - 空间释放
  - 扩展属性删除

阶段4: 提交阶段
  - 事务提交
  - 日志写入
  - 缓存清理
```

## 二、目录项删除

### 2.1 目录项查找与删除

从父目录中删除目标文件的目录项：

```
1. 查找目录项
   - 计算文件名哈希值
   - 在目录B+树中查找
   ↓
2. 验证匹配
   - 名称完全匹配
   - inode号匹配
   ↓
3. 标记为未使用
   struct xfs_dir2_data_unused {
       __be16  freetag;    /* XFS_DIR2_DATA_FREE_TAG */
       __be16  length;     /* 空闲长度 */
   };
   ↓
4. 更新最佳空闲空间
   - 更新目录块的最佳空闲空间信息
   ↓
5. 可能触发目录收缩
   - 如果空闲空间过多，可能转换格式
```

### 2.2 目录格式维护

删除目录项后维护目录格式：

| 目录格式 | 删除操作 | 可能转换 |
|----------|----------|----------|
| 短格式 | 直接删除 | 保持短格式 |
| 块格式 | 标记空闲 | 可能转短格式 |
| B+树格式 | 标记空闲 | 可能转块格式 |

转换条件：
- 短格式：目录项数=0（删除最后一个）
- 块格式：空闲空间超过阈值
- B+树格式：目录项数低于阈值

### 2.3 目录项删除优化

XFS优化目录项删除：

| 优化 | 描述 | 收益 |
|------|------|------|
| 惰性删除 | 仅标记为未使用 | 快速删除 |
| 空间合并 | 合并相邻空闲空间 | 减少碎片 |
| 延迟格式转换 | 延迟到必要时 | 减少开销 |
| 批量删除 | 合并多个删除 | 提高效率 |

## 三、链接计数管理

### 3.1 链接计数减少

减少文件的硬链接计数：

```c
void xfs_droplink(
    struct xfs_trans *tp,
    struct xfs_inode *ip)
{
    /* 减少链接计数 */
    ip->i_d.di_nlink--;
    
    /* 如果链接数=0 */
    if (ip->i_d.di_nlink == 0) {
        /* 标记inode为删除状态 */
        ip->i_d.di_mode = 0;
        
        /* 计划资源释放 */
        xfs_ifree_queue(ip);
    }
    
    /* 更新时间戳 */
    ip->i_d.di_ctime = current_time;
}
```

### 3.2 链接计数为零的处理

当文件链接计数为零时：

```
链接计数=0
    ↓
标记inode为删除
    ↓
加入ifree队列
    ↓
异步释放资源:
1. 释放数据块
2. 释放扩展属性
3. 释放inode号
    ↓
最终从磁盘删除
```

### 3.3 打开文件处理

文件被删除但仍被打开时的特殊处理（Linux特性）：

```
场景: 进程A打开文件，进程B删除文件

处理:
1. 文件仍然存在磁盘上
2. 链接计数=0
3. inode标记为删除
4. 进程A仍可访问文件
5. 进程A关闭文件时:
   - 如果链接计数=0 → 释放资源
   - 否则 → 保持文件
```

## 四、空间释放

### 4.1 Extent释放流程

释放文件占用的数据块：

```
1. 遍历文件的所有extent
   for each extent in file:
       ↓
2. 减少引用计数
   - 如果引用计数>1 (共享块):
       xfs_refcount_decrease_extent()
   ↓
3. 删除反向映射
   xfs_rmap_delete()
   ↓
4. 释放空间
   xfs_free_extent()
       ↓
   a. 加入空闲B+树
   b. 尝试与相邻空闲extent合并
   c. 更新AGF统计
```

### 4.2 延迟释放

XFS可能延迟释放空间以优化性能：

| 延迟场景 | 原因 | 最终释放时机 |
|----------|------|--------------|
| 繁忙extent | 正在使用 | 使用完成后 |
| 事务中extent | 事务未提交 | 事务提交后 |
| 日志中的extent | 日志未刷写 | 日志检查点后 |
| CoW共享extent | 仍有引用 | 所有引用释放后 |

### 4.3 空间释放优化

优化空间释放性能：

| 优化技术 | 描述 | 收益 |
|----------|------|------|
| 批量释放 | 合并多个extent释放 | 减少B+树操作 |
| 异步释放 | 后台释放资源 | 快速响应用户 |
| 预合并 | 释放时检查相邻空间 | 减少碎片 |
| 惰性更新 | 延迟更新统计信息 | 减少元数据写入 |

## 五、inode释放

### 5.1 inode释放流程

释放不再使用的inode号：

```
1. 标记inode为空闲
   - 清除inode位图中的位
   - 更新AGI统计
   ↓
2. 加入空闲inodeB+树
   - 如果AG支持空闲inodeB+树
   - 否则仅更新位图
   ↓
3. 清理inode缓存
   - 从VFS inode缓存移除
   - 从XFS inode缓存移除
   ↓
4. 释放inode块（可选）
   - 如果inode块完全空闲
   - 释放整个inode块
```

### 5.2 空闲inode管理

XFS管理空闲inode的两种方式：

| 管理方式 | 描述 | 使用场景 |
|----------|------|----------|
| 位图管理 | 传统位图方式 | 默认方式 |
| 空闲inodeB+树 | B+树索引空闲inode | 稀疏inode分配时 |

### 5.3 inode释放优化

优化inode释放和重用：

1. **局部性保留**: 最近释放的inode优先重用
2. **批量释放**: 批量释放inode减少开销
3. **延迟释放**: 延迟实际释放到合适时机
4. **预分配**: 预分配inode块提高分配速度

## 六、扩展属性删除

### 6.1 属性删除流程

删除文件的扩展属性：

```
1. 查找属性
   - 本地属性: 扫描inode内属性列表
   - 远程属性: 查询属性B+树
   ↓
2. 删除属性
   - 本地属性: 从inode内删除
   - 远程属性: 释放属性数据块
   ↓
3. 更新属性B+树
   - 如果属性在B+树中，删除条目
   - 可能触发B+树收缩
   ↓
4. 更新inode
   - 减少属性计数
   - 更新时间戳
```

### 6.2 属性命名空间清理

不同命名空间的属性有不同的清理策略：

| 命名空间 | 删除时机 | 特殊处理 |
|----------|----------|----------|
| user | 文件删除时 | 无条件删除 |
| trusted | 文件删除时 | 需要特权 |
| security | 文件删除时 | 安全模块通知 |
| system | 文件删除时 | 保留某些系统属性 |

### 6.3 大属性删除优化

删除大属性时的优化：

1. **延迟释放**: 大属性块延迟释放
2. **批量删除**: 合并多个属性删除
3. **异步删除**: 后台删除减少延迟
4. **空间回收**: 立即回收空间避免碎片

## 七、目录删除

### 7.1 目录删除特殊性

目录删除需要额外检查：

```
1. 检查目录是否为空
   - 遍历目录项，排除"."和".."
   - 如果非空，返回ENOTEMPTY
   ↓
2. 删除目录项
   - 从父目录删除目录项
   ↓
3. 减少链接计数
   - 目录本身链接计数-1
   - 父目录链接计数-1（删除".."引用）
   ↓
4. 释放目录资源
   - 释放目录数据块
   - 释放目录inode
```

### 7.2 目录遍历检查

检查目录是否为空的优化方法：

| 检查方法 | 描述 | 效率 |
|----------|------|------|
| 完全遍历 | 遍历所有目录项 | O(n) |
| 快速检查 | 检查目录大小/格式 | O(1) |
| 惰性检查 | 信任缓存，实际删除时验证 | 平均O(1) |

XFS使用惰性检查：
1. 快速检查目录格式和大小
2. 如果可能非空，进行完整遍历
3. 实际删除时再次验证

### 7.3 目录资源释放

释放目录特有资源：

1. **目录块释放**: 释放所有目录数据块
2. **目录B+树释放**: 如果使用B+树，释放所有节点
3. **目录扩展属性**: 删除目录特有属性
4. **ACL释放**: 删除目录的访问控制列表

## 八、错误处理与恢复

### 8.1 删除失败场景

| 失败原因 | 错误码 | 处理方式 |
|----------|--------|----------|
| 权限不足 | EACCES | 中止删除 |
| 文件不存在 | ENOENT | 提前返回 |
| 目录非空 | ENOTEMPTY | 中止删除 |
| 文件忙 | EBUSY | 强制删除或中止 |
| 磁盘错误 | EIO | 回滚事务 |

### 8.2 事务回滚

删除操作失败时的回滚：

```
删除失败
    ↓
xfs_trans_cancel()  ← 取消事务
    ↓
恢复已修改资源:
1. 恢复目录项（如果已删除）
2. 恢复链接计数
3. 恢复时间戳
    ↓
清理临时状态
    ↓
返回错误码
```

### 8.3 崩溃恢复

系统崩溃后的删除恢复：

```
崩溃后恢复:
1. 扫描日志，找到未完成的删除操作
2. 重做已提交的删除操作
3. 撤销未提交的删除操作
4. 验证文件系统一致性
```

## 九、性能优化

### 9.1 批量删除优化

批量删除文件时的优化：

| 优化技术 | 描述 | 收益 |
|----------|------|------|
| 事务合并 | 多个删除合并为单个事务 | 减少日志写入 |
| 批量空间释放 | 合并extent释放操作 | 减少B+树操作 |
| 异步资源释放 | 后台释放资源 | 快速响应用户 |
| 缓存预热 | 预加载相关元数据 | 减少磁盘IO |

### 9.2 并发删除处理

多进程并发删除时的锁管理：

```c
/* 删除操作的锁顺序 */
1. xfs_ilock(parent, XFS_ILOCK_EXCL);  /* 锁父目录 */
2. xfs_ilock(target, XFS_ILOCK_EXCL);  /* 锁目标文件 */
3. 执行删除操作
4. xfs_iunlock(target, XFS_ILOCK_EXCL);
5. xfs_iunlock(parent, XFS_ILOCK_EXCL);
```

死锁避免策略：
- 固定锁顺序：父目录→子文件
- 超时机制：锁等待超时
- 死锁检测：检测循环等待

### 9.3 资源回收优化

优化资源回收性能：

1. **增量回收**: 分批次回收资源，避免卡顿
2. **优先级回收**: 先回收大资源，再回收小资源
3. **后台回收**: 低优先级后台回收
4. **智能合并**: 智能合并空闲空间
