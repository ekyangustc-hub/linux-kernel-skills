# XFS Reference Count B+tree (引用计数B+树)

## 一、概述

### 1.1 引用计数概念

引用计数B+树（refc btree）用于跟踪共享数据块的引用情况，支持写时复制（Copy-on-Write, CoW）操作：

```
文件A: 块100-200 ←──────────┐
                              │ 共享
文件B: 块300-400 ───────→ 数据块[100-200]
                              ↑
文件C: 块500-600 ────────────┘
     引用计数 = 3
```

### 1.2 应用场景

| 场景 | 描述 | 引用计数变化 |
|------|------|--------------|
| 写时复制 | 共享数据块，直到修改才复制 | 创建时增加，修改时减少 |
| 文件克隆 | 快速创建文件副本 | 立即增加引用 |
| 快照 | 创建文件系统快照 | 快照间共享数据块 |
| 重命名 | 文件移动 | 引用不变 |

### 1.3 引用计数范围

```
struct xfs_refcount_rec {
    __be32  rc_startblock;  /* 起始块号 */
    __be32  rc_blockcount;  /* 块数量 */
    __be32  rc_refcount;    /* 引用计数 */
};
```

引用计数值范围：1-2^32-1

## 二、引用计数B+树结构

### 2.1 B+树布局

引用计数B+树存储在AG的元数据区域：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Reference Count B+tree                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Root Node                                                        │   │
│  │   (根节点，存储在AGF中)                                              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Internal Nodes                                                   │   │
│  │   (内部节点，多级索引)                                               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Leaf Nodes                                                       │   │
│  │   (叶节点，存储实际引用记录)                                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 节点结构

#### 2.2.1 节点头
```c
struct xfs_btree_block {
    __be32          bb_magic;     /* 魔数: REFC */
    __be16          bb_level;     /* 树层级 */
    __be16          bb_numrecs;   /* 记录数 */
    __be32          bb_leftsib;   /* 左兄弟指针 */
    __be32          bb_rightsib;  /* 右兄弟指针 */
};
```

#### 2.2.2 内部节点记录
```c
struct xfs_refcount_key {
    __be32          rc_startblock;  /* 起始块号 */
};

struct xfs_refcount_ptr {
    __be32          rc_blockcount;  /* 子节点块数 */
};
```

#### 2.2.3 叶节点记录
```c
struct xfs_refcount_rec {
    __be32          rc_startblock;  /* 起始块号 */
    __be32          rc_blockcount;  /* 块数量 */
    __be32          rc_refcount;    /* 引用计数 */
};
```

### 2.3 键值排序

记录按 `rc_startblock` 升序排序，便于范围查询：

```
叶节点:
[块100, 10块, 引用3] → [块200, 5块, 引用2] → [块300, 20块, 引用1]
```

## 三、引用计数操作

### 3.1 增加引用 (Increment)

当数据块被共享时增加引用计数：

```
1. 查找块范围 [start, start+count)
   ↓
2. 如果找到匹配记录:
   - 更新现有记录的引用计数
   ↓
3. 如果未找到:
   - 创建新记录，引用计数=1
   ↓
4. 可能触发记录合并:
   - 相邻块+相同引用计数 → 合并
   ↓
5. 更新B+树
```

### 3.2 减少引用 (Decrement)

当共享关系解除时减少引用计数：

```
1. 查找块范围 [start, start+count)
   ↓
2. 减少引用计数
   ↓
3. 如果引用计数=0:
   - 删除记录
   - 块变为空闲状态
   ↓
4. 可能触发记录分裂:
   - 部分块引用减少 → 分裂记录
   ↓
5. 更新B+树
```

### 3.3 查询引用计数

查询特定块的引用状态：

```
1. 在B+树中查找包含目标块的记录
   ↓
2. 如果找到:
   - 返回引用计数值
   ↓
3. 如果未找到:
   - 返回引用计数=0 (未分配)
   - 或引用计数=1 (非共享块)
```

## 四、写时复制 (CoW) 机制

### 4.1 CoW基本原理

写时复制延迟数据复制，直到真正需要修改时：

```
原始状态:
文件A ──→ 数据块[100-200] (引用=1)
      复制引用
文件B ──→ 数据块[100-200] (引用=2)

文件B写入:
文件B修改块150 → 需要实际复制
1. 分配新块: 块500
2. 复制数据: 块100-200 → 块500
3. 更新引用: 块100-200引用=1，块500引用=1
4. 更新映射: 文件B指向块500
```

### 4.2 CoW操作流程

```
1. 准备写入共享块
   ↓
2. 检查引用计数
   - 引用=1 → 直接写入
   - 引用>1 → 需要CoW
   ↓
3. CoW操作:
   a. 分配新块
   b. 复制数据
   c. 减少原块引用计数
   d. 设置新块引用计数=1
   ↓
4. 更新文件块映射
   ↓
5. 执行实际写入
```

### 4.3 CoW优化策略

| 策略 | 描述 | 优点 |
|------|------|------|
| 批量CoW | 多个块一起复制 | 减少随机IO |
| 预分配 | 预先分配CoW空间 | 避免碎片 |
| 延迟释放 | 延迟释放原块 | 支持回滚 |

## 五、引用计数与空间管理

### 5.1 引用计数与空闲空间

引用计数影响空间管理决策：

| 引用计数 | 空间状态 | 管理策略 |
|----------|----------|----------|
| 0 | 空闲块 | 可分配 |
| 1 | 独占块 | 可直接修改 |
| >1 | 共享块 | 需要CoW才能修改 |

### 5.2 引用计数与RMAP B+树

引用计数B+树与RMAP B+树协同工作：

```
RMAP B+树: 块 → 所有者列表
    例如: 块100 → [文件A, 文件B, 文件C]

Refcount B+树: 块范围 → 引用计数
    例如: 块100-200 → 引用=3

关系: RMAP记录数 ≈ Refcount计数值
```

### 5.3 引用计数维护

引用计数在以下操作中更新：

| 操作 | 引用计数变化 | 触发条件 |
|------|--------------|----------|
| 文件克隆 | 增加 | 创建文件副本 |
| 快照创建 | 增加 | 创建快照 |
| 写时复制 | 先增后减 | 修改共享块 |
| 文件删除 | 减少 | 删除文件 |
| 快照删除 | 减少 | 删除快照 |

- `xfs_refcount_decrease_extent()`: 减少引用计数
