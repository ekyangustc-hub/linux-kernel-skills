# XFS Journal Disk Structure (日志磁盘结构)

## 一、概述

### 1.1 XFS日志作用

XFS使用写前日志（Write-Ahead Logging, WAL）确保文件系统一致性：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              XFS Journal Architecture                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Metadata Operation                    │   Data Operation          │   │
│  │   (inode更新、目录修改等)                 │   (文件内容写入)          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                     Transaction Layer                               │   │
│  │                    (事务层，生成日志记录)                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                     Journal (Log)                                  │   │
│  │          (先写入日志，保证操作可恢复)                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                     Actual Filesystem                              │   │
│  │                 (后更新实际文件系统)                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 1.2 日志位置

| 日志类型 | 位置 | 说明 |
|----------|------|------|
| 内部日志 | 文件系统内部 | 在数据设备上分配区域 |
| 外部日志 | 单独设备 | 专用日志设备 |
| 实时日志 | 实时设备部分 | 与实时设备结合 |

## 二、日志磁盘布局

### 2.1 整体布局

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                              Journal Layout                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Log Header                                                        │   │
│  │   (日志头，包含元数据)                                                │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Cycle 1 Records                                                   │   │
│  │   (周期1的日志记录)                                                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Log Footer                                                        │   │
│  │   (日志尾，标记周期结束)                                              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Cycle 2 Records                                                   │   │
│  │   (周期2的日志记录)                                                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Log Footer                                                        │   │
│  │   (日志尾)                                                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ... 循环使用                                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 日志循环机制

XFS日志采用循环写入机制：

```
周期 1: [头][记录1][记录2]...[尾]
周期 2: [头][记录3][记录4]...[尾]
周期 3: [头][记录5][记录6]...[尾]
...
```

当日志写满时，覆盖最旧的记录（如果已提交）。

## 三、日志记录格式

### 3.1 日志记录结构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                            Log Record Format                               │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Log Record Header (xlog_rec_header)                               │   │
│  │   - h_magicno: 魔数 (0x554c4f47 "XLOG")                              │   │
│  │   - h_cycle: 周期号                                                   │   │
│  │   - h_version: 版本                                                   │   │
│  │   - h_len: 数据长度                                                   │   │
│  │   - h_clientid: 客户端ID                                              │   │
│  │   - h_fmt: 格式                                                       │   │
│  │   - h_size: 头大小                                                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Log Item Header (xlog_op_header)                                  │   │
│  │   - oh_clientid: 客户端ID                                            │   │
│  │   - oh_res2: 保留                                                    │   │
│  │   - oh_flags: 标志                                                   │   │
│  │   - oh_len: 数据长度                                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Item Data                                                         │   │
│  │   (实际元数据内容)                                                    │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   CRC Checksum                                                      │   │
│  │   (CRC校验，确保数据完整性)                                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 客户端ID类型

| 客户端ID | 客户端名称 | 描述 |
|----------|------------|------|
| XFS_TRANSACTION | 事务系统 | 事务管理 |
| XFS_LOG | 日志系统 | 日志管理 |
| XFS_INODE | inode操作 | inode更新 |
| XFS_DIR | 目录操作 | 目录修改 |
| XFS_EXTENT | extent操作 | extent分配 |
| XFS_BTREE | B+树操作 | B+树更新 |
| XFS_ATTR | 属性操作 | 属性修改 |

### 3.3 日志记录类型

| 记录类型 | 操作 | 内容 |
|----------|------|------|
| 创建记录 | 创建文件/目录 | inode分配、目录项 |
| 删除记录 | 删除文件/目录 | inode释放、目录项删除 |
| 重命名记录 | 重命名 | 目录项更新 |
| 截断记录 | 文件截断 | extent更新 |
| 属性记录 | 属性修改 | 属性数据 |

## 四、检查点机制

### 4.1 检查点作用

检查点将内存中的脏数据同步到磁盘，并清理日志：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Checkpoint Process                                 │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  1. 暂停新事务                                                              │
│  2. 等待当前事务完成                                                        │
│  3. 将CIL (Commit Item List) 写入日志                                      │
│  4. 将日志记录标记为检查点                                                  │
│  5. 将脏数据写入实际文件系统                                                │
│  6. 更新日志尾                                                              │
│  7. 清理已提交的日志记录                                                    │
│  8. 恢复事务处理                                                            │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.2 CIL (Commit Item List)

CIL是XFS的关键优化，将多个事务合并为单个日志记录：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         Commit Item List (CIL)                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Transaction 1                                                    │   │
│  │   - inode更新                                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Transaction 2                                                    │   │
│  │   - 目录项添加                                                       │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Transaction 3                                                    │   │
│  │   - extent分配                                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  合并为单个日志记录 → 写入磁盘                                              │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 五、恢复机制

### 5.1 日志恢复流程

系统崩溃后，XFS通过日志恢复一致性：

```
1. 读取日志头，找到最新有效记录
   ↓
2. 向前扫描日志，重建事务列表
   ↓
3. 重做 (Redo) 已提交但未写入磁盘的操作
   ↓
4. 撤销 (Undo) 未提交的操作
   ↓
5. 验证文件系统一致性
   ↓
6. 更新超级块，标记恢复完成
```

### 5.2 恢复阶段

| 阶段 | 操作 | 目的 |
|------|------|------|
| 日志扫描 | 读取日志记录 | 重建操作序列 |
| 重做阶段 | 应用已提交操作 | 恢复已完成操作 |
| 撤销阶段 | 回滚未提交操作 | 清理部分完成操作 |
| 验证阶段 | 检查文件系统 | 确保一致性 |

## 六、内核源码参考

### 6.1 主要数据结构

- `xlog_rec_header`: 日志记录头
- `xlog_op_header`: 操作头
- `xlog_in_core`: 内存中日志结构
- `xlog_t`: 日志控制结构
- `xfs_log_item`: 日志项基类

### 6.2 日志客户端

- `xfs_trans_t`: 事务结构
- `xfs_log_item_desc_t`: 日志项描述符
- `xfs_buf_log_item_t`: 缓冲区日志项
- `xfs_inode_log_item_t`: inode日志项
- `xfs_efi_log_item_t`: 扩展项日志项

### 6.3 关键函数

- `xlog_write()`: 写入日志
- `xlog_commit()`: 提交日志记录
- `xlog_recover()`: 日志恢复
- `xlog_checkpoint()`: 检查点
- `xlog_grant_head_wait()`: 日志空间等待

### 6.4 文件位置

- `fs/xfs/xfs_log.h`: 日志结构定义
- `fs/xfs/xfs_log_recover.c`: 日志恢复
- `fs/xfs/xfs_log_priv.h`: 日志私有结构
- `fs/xfs/xfs_trans.h`: 事务管理
- `fs/xfs/xfs_trans_ail.c`: AIL (Active Item List) 管理
