# XFS Reverse Mapping B+tree (反向映射B+树)

## 一、概述

### 1.1 反向映射概念

反向映射B+树（rmap btree）建立从物理块到所有者的映射关系，是传统正向映射（文件→块）的逆关系：

```
正向映射 (文件到块):
文件A → 块[100, 150, 200]
文件B → 块[120, 180, 220]

反向映射 (块到文件):
块100 → 文件A
块120 → 文件B
块150 → 文件A
块180 → 文件B
块200 → 文件A
块220 → 文件B
```

### 1.2 反向映射的作用

| 功能 | 描述 | 传统方法问题 |
|------|------|--------------|
| 精确空间回收 | 知道每个块的所有者 | 需要扫描全盘 |
| 碎片整理 | 识别碎片位置 | 难以确定碎片关系 |
| 写时复制 | 快速找到共享块 | 需要遍历所有文件 |
| 崩溃恢复 | 重建文件系统状态 | 恢复复杂 |

### 1.3 反向映射记录格式

```c
struct xfs_rmap_rec {
    __be32          rm_startblock;  /* 起始块号 */
    __be32          rm_blockcount;  /* 块数量 */
    __be64          rm_owner;       /* 所有者 */
    __be64          rm_offset;      /* 文件内偏移 */
    __be32          rm_flags;       /* 标志位 */
};
```

## 二、反向映射B+树结构

### 2.1 B+树布局

反向映射B+树存储在AG的元数据区域：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Reverse Mapping B+tree                                  │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Root Node                                                        │   │
│  │   (根节点，存储在AGF中)                                              │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Internal Nodes                                                   │   │
│  │   (内部节点，多级索引)                                               │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                      ↓                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │   Leaf Nodes                                                       │   │
│  │   (叶节点，存储实际反向映射记录)                                      │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 所有者标识

所有者字段编码所有者类型和ID：

```
所有者 (rm_owner):
┌──────────────┬──────────────┐
│   类型 (4位)  │    ID (60位)  │
└──────────────┴──────────────┘

所有者类型:
XFS_RMAP_OWN_UNKNOWN    = 0   /* 未知 */
XFS_RMAP_OWN_INODES     = 1   /* inode块 */
XFS_RMAP_OWN_LOG        = 2   /* 日志 */
XFS_RMAP_OWN_AG         = 3   /* AG元数据 */
XFS_RMAP_OWN_INOBT      = 4   /* inode B+树 */
XFS_RMAP_OWN_REFC       = 5   /* 引用计数B+树 */
XFS_RMAP_OWN_COW        = 6   /* CoW分配 */
XFS_RMAP_OWN_FS         = 7   /* 文件系统结构 */
XFS_RMAP_OWN_NULL       = 8   /* 空闲块 */
```

### 2.3 标志位

标志位表示块的特殊属性：

```c
#define XFS_RMAP_ATTR_FORK     (1 << 0)  /* 属性fork块 */
#define XFS_RMAP_BMBT_BLOCK    (1 << 1)  /* B+树块 */
#define XFS_RMAP_UNWRITTEN     (1 << 2)  /* 未写入块 */
#define XFS_RMAP_OFFSET_VALID  (1 << 3)  /* 偏移有效 */
```

## 三、反向映射操作

### 3.1 添加反向映射

当分配新块时添加反向映射记录：

```
1. 构建反向映射记录:
   - rm_startblock: 起始块号
   - rm_blockcount: 块数量
   - rm_owner: 所有者
   - rm_offset: 文件内偏移
   - rm_flags: 标志位
   ↓
2. 在B+树中插入记录
   ↓
3. 可能触发B+树分裂
   ↓
4. 更新AGF中的根指针
```

### 3.2 删除反向映射

当释放块时删除反向映射记录：

```
1. 查找匹配的反向映射记录
   ↓
2. 验证记录匹配:
   - 相同所有者
   - 相同偏移 (如果XFS_RMAP_OFFSET_VALID)
   - 相同标志位
   ↓
3. 从B+树中删除记录
   ↓
4. 可能触发B+树合并
   ↓
5. 更新AGF中的根指针
```

### 3.3 查询反向映射

查询特定块的所有者信息：

```
1. 在B+树中查找包含目标块的记录
   ↓
2. 返回所有匹配记录
   ↓
3. 可能结果:
   - 无记录 → 空闲块
   - 单条记录 → 独占块
   - 多条记录 → 共享块
```

### 3.4 范围查询

查询特定范围内的所有反向映射：

```
1. 设置查询范围 [start, end]
   ↓
2. 从B+树中遍历该范围内的所有记录
   ↓
3. 过滤记录:
   - 按所有者类型
   - 按标志位
   ↓
4. 返回记录列表
```

## 四、反向映射应用场景

### 4.1 精确空间回收

使用反向映射精确回收空间，无需扫描全盘：

```
传统方法:
1. 扫描所有inode的extent列表
2. 构建块分配位图
3. 对比空闲位图，找出未引用块

反向映射方法:
1. 查询反向映射B+树
2. 直接获得每个块的所有者
3. 验证所有者是否有效
4. 回收无主块
```

### 4.2 碎片整理

识别文件碎片，进行整理：

```
1. 查询文件的所有块
   ↓
2. 通过反向映射找到物理位置
   ↓
3. 分析碎片情况:
   - 连续块数
   - 块间隔距离
   ↓
4. 选择需要整理的碎片
   ↓
5. 重新分配连续空间
   ↓
6. 更新正向和反向映射
```

### 4.3 写时复制支持

快速找到共享块，支持CoW：

```
1. 文件要修改块X
   ↓
2. 查询反向映射: 块X → [文件A, 文件B]
   ↓
3. 如果引用数>1:
   a. 分配新块Y
   b. 复制数据: X → Y
   c. 更新文件A映射: X → Y
   d. 更新反向映射:
      - 减少块X引用
      - 添加块Y映射
```

### 4.4 崩溃恢复

使用反向映射加速恢复：

```
1. 扫描磁盘所有块
   ↓
2. 查询每个块的反向映射
   ↓
3. 验证映射一致性:
   - 正向映射 vs 反向映射
   - 引用计数 vs 反向映射
   ↓
4. 修复不一致的映射
```

## 五、反向映射与引用计数的关系

### 5.1 协同工作

反向映射和引用计数B+树协同管理共享块：

```
场景: 块100被文件A和文件B共享

反向映射B+树:
块100 → [文件A, 文件B] (2条记录)

引用计数B+树:
块100-100 → 引用计数=2

关系: 反向映射记录数 = 引用计数值
```

### 5.2 一致性维护

两个B+树必须保持一致性：

| 操作 | 反向映射更新 | 引用计数更新 |
|------|--------------|--------------|
| 分配新块 | 添加记录 | 设置计数=1 |
| 共享块 | 添加记录 | 增加计数 |
| 释放块 | 删除记录 | 减少计数 |
| CoW操作 | 更新记录 | 先增后减 |

### 5.3 验证机制

定期验证两个B+树的一致性：

```
1. 遍历引用计数B+树
   ↓
2. 对每个引用记录:
   a. 查询反向映射中的对应块
   b. 比较记录数量 vs 引用计数值
   ↓
3. 报告不一致项
   ↓
4. 启动修复流程
```

## 六、内核源码参考

### 6.1 主要数据结构

- `xfs_rmap_rec`: 反向映射记录
- `xfs_rmap_key`: 反向映射键
- `xfs_rmap_ptr`: 反向映射指针
- `xfs_rmap_irec`: 内存中反向映射记录
- `xfs_rmap_intent`: 反向映射操作意图

### 6.2 关键函数

- `xfs_rmap_lookup_le()`: 查找小于等于键的记录
- `xfs_rmap_get_rec()`: 获取反向映射记录
- `xfs_rmap_insert()`: 插入反向映射记录
- `xfs_rmap_delete()`: 删除反向映射记录
- `xfs_rmap_compare()`: 比较反向映射记录
- `xfs_rmap_finish_one()`: 完成反向映射操作

### 6.3 文件位置

- `fs/xfs/xfs_rmap.h`: 反向映射结构定义
- `fs/xfs/xfs_rmap.c`: 反向映射核心实现
- `fs/xfs/xfs_rmap_btree.c`: 反向映射B+树操作
- `fs/xfs/xfs_rmap_item.c`: 反向映射日志项

### 6.4 所有者类型定义

- `fs/xfs/xfs_rmap.h`: 所有者类型常量
- `fs/xfs/xfs_rmap.c`: 所有者编码/解码函数
- `fs/xfs/xfs_rmap_btree.c`: 所有者比较函数
